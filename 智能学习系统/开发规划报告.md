## 二、接下来的开发步骤（按优先级排序）

### 步骤 1：数据初始化（当前最优先）

**目标**：让数据库有真实可用的数据，用户登录后能看到内容。

**具体操作**：
1. **创建数据初始化脚本** `backend/init_data.py`
   - 插入 1-2 个测试用户
   - 插入各学科的学习资料（数学、英语、物理等，每个学科 5-10 条）
   - 插入示例错题（每个学科 3-5 条）
   - 插入示例练习题（选择题、填空题等）
   - 插入默认成就定义（初学者、练习达人、学霸等 8-10 个）
   - 插入示例学习路径（高考冲刺、期末复习等 2-3 个）

2. **运行初始化脚本**
   ```bash
   cd backend
   python init_data.py
   ```

3. **验证数据**
   - 登录后检查各页面是否有数据展示

---

### 步骤 2：前端与后端 API 联调（核心工作）

**目标**：确保每个前端页面能正确调用后端 API，数据能正常显示和交互。

**联调顺序**（按页面依赖关系）：

| 顺序 | 页面 | 需验证的 API | 验收标准 |
|-----|------|------------|---------|
| 1 | 登录/注册 | POST /auth/login, POST /auth/register | 登录成功跳转首页，Token 正确存储 |
| 2 | 个人档案 | GET /auth/me, PUT /auth/profile | 显示用户信息，可修改资料 |
| 3 | 学习资料库 | GET /study-materials, POST /study-materials | 列表有数据，可添加资料 |
| 4 | 错题题库 | GET /mistakes, POST /mistakes | 列表有数据，可添加错题 |
| 5 | 练习测试 | GET /exercises, POST /exercises/{id}/submit | 可答题，提交后显示结果 |
| 6 | 学习路径 | GET /learning-paths, POST /learning-paths | 可创建路径，标记步骤完成 |
| 7 | 我的收藏 | GET /favorites, POST /favorites | 可收藏/取消收藏 |
| 8 | 学习记录 | GET /timeline, GET /timeline/stats/overview | 显示学习统计和时间线 |
| 9 | 成就排行 | GET /achievements, GET /achievements/leaderboard | 显示成就和排行榜 |
| 10 | 通知 | GET /notifications, PUT /notifications/read-all | 显示通知，可标记已读 |
| 11 | 课堂互动 | GET /classrooms, POST /classrooms | 可创建/加入课堂 |
| 12 | 思维导图 | POST /mindmaps/generate | 调用大模型生成思维导图 |
| 13 | 成绩分析 | GET /analytics/overview | 显示学习统计数据 |

**联调方法**：
1. 打开浏览器开发者工具（F12）
2. 切换到 Network 标签
3. 操作页面功能，观察 API 请求和响应
4. 如有报错，根据错误信息修复前端请求或后端接口

---

### 步骤 3：大模型智能生成功能验证

**目标**：确保调用讯飞星火/火山方舟 API 的功能正常工作。

**需验证的功能**：
- 思维导图生成（POST /mindmaps/generate）
- 成绩评估报告生成（POST /analytics/report/generate）
- 课堂 PPT 生成（POST /classrooms/{id}/ppt）
- 智能对话（POST /chat）

**验证步骤**：
1. 确认 `.env` 中 API Key 配置正确
2. 在前端触发对应功能
3. 观察后端日志，确认有 API 调用
4. 检查返回结果是否合理

---

### 步骤 4：前端细节优化

**目标**：提升用户体验，修复视觉和交互问题。

**优化内容**：
1. **空状态处理**：无数据时显示引导文案（如"暂无错题，点击添加"）
2. **加载状态**：API 请求时显示 Loading 动画
3. **错误提示**：API 失败时给用户友好提示
4. **响应式布局**：确保在不同屏幕尺寸下正常显示
5. **清理 TypeScript 警告**：删除未使用的变量和导入

---

### 步骤 5：端到端场景测试

**目标**：模拟真实用户完整使用流程。

**测试场景**：
1. **新用户注册 → 完善资料 → 浏览学习资料 → 收藏资料 → 制定学习路径 → 完成练习 → 查看成绩分析**
2. **老师创建课堂 → 学生加入 → 发起投票 → 参与测验 → 查看结果**
3. **学生做错题 → 加入错题本 → 复习错题 → 标记已掌握**

---

### 步骤 6：性能优化与部署准备

**目标**：确保系统稳定运行，可对外演示。

**优化内容**：
1. 后端 API 响应时间优化（添加数据库索引）
2. 前端首屏加载优化（代码分割、懒加载）
3. 配置生产环境变量
4. 编写部署文档

## 六、总结

**你的分析基本正确，但顺序需要调整：**

1. **不是"先前端设计好再接后端"** — 前端页面已经都写好了，现在需要的是**联调**（让前端页面真正调用后端 API 获取数据）
2. **不是"搞数据库"** — 数据库表已经都创建好了，现在需要的是**填充初始数据**
3. **正确的顺序是**：填充数据 → 前后端联调 → 大模型验证 → 细节优化 → 端到端测试

**当前最优先做的事**：
> 编写数据初始化脚本，让数据库有真实内容，然后逐个页面前后端联调。
