# 开发进度报告

**报告日期：** 2026年6月19日  
**项目名称：** 基于大模型的个性化资源生成与学习多智能体系统  
**赛题定位：** 中国软件杯大赛 A3 赛题

---

## 一、项目目标与实现策略

### 1.1 核心目标

构建基于大模型的多智能体学习系统，实现：
- **个性化学习资源自动生成**（课件、习题、思维导图、复习讲义）
- **动态学生学习画像**（诊断-生成-练习-复盘完整闭环）
- **六大智能体协同工作**（学情诊断、路径规划、资源生成、习题批改、资源优化、人机交互）

### 1.2 技术架构

```
┌─────────────────────────────────────────────────────────────┐
│  用户交互层（React + TypeScript + Tailwind CSS + Ant Design）│
├─────────────────────────────────────────────────────────────┤
│  多智能体调度中枢（FastAPI + RabbitMQ + 任务队列）           │
├─────────────────────────────────────────────────────────────┤
│  大模型能力底座（讯飞星火 API + Chroma + FAISS + BGE）       │
├─────────────────────────────────────────────────────────────┤
│  数据持久层（SQLite + SQLAlchemy + Redis）                  │
└─────────────────────────────────────────────────────────────┘
```

### 1.3 任务分工

| 角色 | 职责 | 负责模块 |
|------|------|---------|
| **架构师** | 技术选型、系统设计、智能体调度 | 核心框架、多智能体体系 |
| **后端开发** | API接口开发、数据库设计、智能体实现 | 所有API接口、业务逻辑 |
| **前端开发** | 页面开发、用户交互、可视化展示 | 所有页面组件、UI交互 |
| **AI工程师** | 大模型集成、提示词工程、RAG实现 | LLM调用、资源生成、智能体逻辑 |
| **测试工程师** | 功能测试、性能测试、效果验证 | 测试用例、试点验证 |

## 六、风险与依赖

### 6.1 技术风险

| 风险 | 描述 | 影响 | 缓解措施 |
|------|------|------|---------|
| 大模型API稳定性 | 外部API调用可能失败 | 资源生成功能不可用 | 增加重试机制、降级方案 |
| 多智能体调度复杂度 | 智能体间通信可能出现死锁 | 系统响应缓慢或卡死 | 引入消息队列、超时机制 |
| 数据量增长 | 学习数据积累导致查询变慢 | 性能下降 | 数据库索引优化、分页查询 |

### 6.2 外部依赖

| 依赖 | 类型 | 状态 | 说明 |
