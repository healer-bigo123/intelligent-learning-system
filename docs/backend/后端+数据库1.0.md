# 项目概述

本项目是一个**基于大模型的个性化资源生成与学习多智能体系统**（SmartLearning-MAS），后端采用 **FastAPI** 框架构建，核心能力包括：

- **AI 对话**：接入火山方舟 DeepSeek API，支持流式/非流式对话，并基于 RAG（检索增强生成）引用知识库内容作答。
- **知识库管理**：支持 `.txt`、`.md`、`.pdf` 文档上传、自动分块、向量化存储（Chroma / FAISS / 简化版 SQLite）。
- **多智能体调度**：定义了 6 类学习智能体（画像分析师、资源设计师、题库生成师、多媒体制作师、代码导师、路径规划师），提供任务派发接口。
- **数据持久化**：使用 SQLite + SQLAlchemy 存储学生画像、对话记录、学习资源、学习路径、智能体任务等核心数据。

技术栈关键词：FastAPI、Pydantic、SQLAlchemy、SQLite、Chroma、FAISS、LangChain、DeepSeek（火山方舟）、SSE 流式输出。

# 核心模块详解

## 1. 配置管理 — `app/core/config.py`

**功能**：使用 Pydantic Settings 统一管理环境变量与默认配置，自动从 `.env` 文件加载。

**关键类**：
- `Settings(BaseSettings)`：全局配置类，包含应用基础、LLM API、数据库、向量库、Embedding、知识库、多智能体、日志等配置项。
- `settings = Settings()`：全局单例实例。

**技术选型**：`pydantic-settings`

---

## 2. 大模型客户端 — `app/core/llm_client.py`

**功能**：统一封装火山方舟 DeepSeek API，提供对话、流式对话、联网搜索能力。

**关键类**：
- `VolcesLLMClient`
  - `__init__()`：从 `settings` 读取 `VOLCES_API_KEY`、`VOLCES_BASE_URL`、`VOLCES_MODEL`
  - `chat(messages, stream, temperature, max_tokens, tools)`：非流式对话，返回 JSON
  - `chat_stream(messages, ...)`：流式对话（SSE 格式），`AsyncGenerator[str, None]`
  - `generate_with_web_search(query, max_keyword)`：调用 `/responses` 接口，启用 `web_search` 工具
  - `build_messages(system_prompt, user_prompt, history)`：构建标准消息列表

**全局实例**：`llm_client = VolcesLLMClient()`

**技术选型**：`httpx`（异步 HTTP 客户端）

---

## 3. 知识库管理 — `app/core/knowledge_base.py`

**功能**：负责文档加载、文本分块、向量入库、RAG 检索上下文拼接。

**关键类**：
- `KnowledgeBaseManager`
  - `load_document(file_path)`：根据扩展名选择 `TextLoader` / `PyPDFLoader` / `UnstructuredMarkdownLoader`
  - `load_directory(directory_path)`：递归加载目录下所有支持格式的文档
  - `split_documents(documents)`：使用 `RecursiveCharacterTextSplitter` 按 `CHUNK_SIZE`（默认 512）和 `CHUNK_OVERLAP`（默认 50）分块
  - `add_to_knowledge_base(documents, store_type)`：分块后写入 Chroma 或 FAISS
  - `search_knowledge_base(query, k, store_type)`：向量相似度检索
  - `get_context_for_rag(query, k)`：将检索结果格式化为 `[参考N] 来源: ...\n内容...` 的上下文文本

**全局实例**：`knowledge_base_manager = KnowledgeBaseManager()`

**技术选型**：`langchain-community`（文档加载器）、`langchain-text-splitters`（文本分割）

---

## 4. 向量数据库（完整版）— `app/core/vector_store.py`

**功能**：支持 Chroma 和 FAISS 两种向量存储，使用 HuggingFace Embedding 模型。

**关键类**：
- `VectorStoreManager`
  - `init_embedding_model()`：加载 `BAAI/bge-large-zh-v1.5` 模型，支持 `cpu` 设备
  - `init_chroma()`：初始化/加载持久化 Chroma 向量库
  - `init_faiss()`：加载本地 FAISS 索引，若不存在则延迟创建
  - `add_documents_to_chroma(documents)` / `add_documents_to_faiss(texts, metadatas)`：添加文档
  - `search_chroma(query, k)` / `search_faiss(query, k)`：相似度搜索，返回含 `content`、`metadata`、`score` 的字典列表
  - `get_collection_stats()`：返回 Chroma 和 FAISS 的文档数量统计

**全局实例**：`vector_store_manager = VectorStoreManager()`

**技术选型**：`langchain-chroma`、`langchain-community(FAISS)`、`langchain-huggingface(HuggingFaceEmbeddings)`、`sentence-transformers`

---

## 5. 简化版向量存储 — `app/core/simple_vector_store.py`

**功能**：在完整版向量库依赖缺失时的降级方案，基于 SQLite + 全文搜索（FTS5）或 LIKE 查询。

**关键类**：
- `SimpleVectorStore`
  - `_init_db()`：创建 `documents` 表，尝试创建 `documents_fts` 虚拟表及同步触发器
  - `add_document(content, source, metadata)`：插入文档
  - `search(query, k)`：优先使用 FTS5 全文搜索，不可用时回退到 `LIKE '%query%'`
  - `get_stats()`：返回文档总数
  - 提供与 `VectorStoreManager` 兼容的接口：`add_documents_to_chroma`、`search_chroma`、`search_faiss`、`get_collection_stats`

**全局实例**：`simple_vector_store = SimpleVectorStore()`

**技术选型**：内置 `sqlite3`（FTS5 可选）

---

## 6. 数据库 ORM — `app/models/database.py`

**功能**：定义所有业务数据模型，管理 SQLite 数据库连接与会话。

**关键对象**：
- `engine`：SQLAlchemy 引擎，`create_engine(settings.DATABASE_URL)`
- `SessionLocal`：会话工厂
- `Base = declarative_base()`：ORM 基类
- `init_db()`：调用 `Base.metadata.create_all(bind=engine)` 自动建表
- `get_db()`：FastAPI 依赖注入用的数据库会话生成器

# API 接口清单

## 根路由（`main.py` 直接挂载）

| 路径 | 方法 | 功能描述 |
|------|------|----------|
| `/` | GET | 返回应用名称、版本、状态、文档地址 |
| `/health` | GET | 基础健康检查，返回 `{"status": "healthy"}` |

## Health 模块（`/api/v1/health`）

| 路径 | 方法 | 功能描述 |
|------|------|----------|
| `/` | GET | 基础健康检查 |
| `/ready` | GET | 就绪检查，验证向量存储和 LLM 客户端状态，返回各依赖项是否就绪 |

## Knowledge 模块（`/api/v1/knowledge`）

| 路径 | 方法 | 功能描述 |
|------|------|----------|
| `/search` | POST | 搜索知识库，参数：`query`、`k`（默认 5）、`store_type`（默认 chroma） |
| `/upload` | POST | 上传文档到知识库，支持 `.txt`、`.md`、`.pdf`，参数：`file`、`store_type` |
| `/stats` | GET | 获取知识库统计信息（向量库文档数、支持格式列表） |

## Chat 模块（`/api/v1/chat`）

| 路径 | 方法 | 功能描述 |
|------|------|----------|
| `/stream` | POST | 流式对话接口（SSE），支持 RAG，参数：`message`、`session_id`、`use_rag`、`history` |
| `/` | POST | 非流式对话接口，支持 RAG，返回完整响应文本 |

**请求体模型**：`ChatRequest`
- `message: str` — 用户消息（必填）
- `session_id: Optional[str]` — 会话 ID
- `use_rag: bool = True` — 是否启用检索增强
- `history: Optional[List[dict]] = []` — 历史消息列表

## Agents 模块（`/api/v1/agents`）

| 路径 | 方法 | 功能描述 |
|------|------|----------|
| `/` | GET | 获取所有已注册智能体列表（6 个内置智能体） |
| `/{agent_id}` | GET | 获取指定智能体详情 |
| `/task` | POST | 向指定智能体派发任务，返回任务 ID 和排队状态 |
| `/task/{task_id}` | GET | 查询任务状态（当前为模拟实现） |

**请求体模型**：`AgentTaskRequest`
- `agent_id: str` — 智能体 ID
- `task_type: str` — 任务类型
- `parameters: Optional[Dict] = {}` — 任务参数
- `user_id: Optional[str] = None` — 用户 ID
