## 四、分开电脑协作机制（关键）

### 1. Git 分支策略
- 每人各自 `feature/模块名` 分支
- 每日晚固定时间提 PR 合并到 `develop`，避免冲突

### 2. 接口契约
- 后端 FastAPI 自动生成 Swagger（`http://localhost:8000/docs`）
- 前端依据 Swagger 对接，避免口头约定

### 3. 数据库同步
- SQLite 无法多人共享，每人本地各自运行 `python init_data.py` 初始化
- schema 变更通过 `migrate.py` 脚本同步
- **不要直接改本地 db 文件**

### 4. 每日汇总
- 固定每晚线上同步 30 分钟
- 合并代码 + 次日任务对齐

### 5. 最终汇总（Day 15）
- 三台电脑代码合并到 `main`
- 统一跑一次 `init_data.py` 重置数据库
- 全功能回归测试

## 五、风险提示

- **SQLite 并发**：最终汇总时只能用一份 db，建议 Day 15 指定 A 的环境为基准，B/C 的数据通过 SQL 导出关键表合并
- **LLM 密钥**：三人各自 `.env` 配置，**不要提交到 Git**（确认 `.env` 已在 `.gitignore`）
- **接口不匹配**：B 对接前先查 Swagger，避免按想象调接口
