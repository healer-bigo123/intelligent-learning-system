# 模型层架构设计

## 一、层定位

模型层（Model Layer）是系统的"认知引擎"，负责大语言模型（LLM）的调用管理、多模态内容的理解与生成、以及模型输出的后处理与质量控制。该层向上为智能体层提供统一、可靠的模型能力接口；向下对接第三方模型服务商或私有化部署的模型集群，屏蔽底层差异，实现"一次接入、灵活切换"。

---

## 二、设计目标

1. **多模型统一调度**：支持同时接入多个模型厂商（OpenAI、Anthropic、国内大模型等），按任务类型、成本、效果自动路由。
2. **多模态原生支持**：无缝处理文本、图像、音频、视频的理解与生成需求。
3. **输出质量可控**：通过提示工程、后处理校验、人机协同审核等手段，确保教育场景下的准确性与安全性。
4. **成本与性能平衡**：实现 Token 级成本监控、缓存机制、模型降级策略，避免资源浪费。

## 四、核心组件详解

### 4.1 模型服务网关（Model Gateway）

#### 4.1.1 请求路由（Request Router）
- **职责**：根据任务类型、模型能力评分、成本预算、用户等级，将请求路由至最优模型。
- **路由策略**：
  - **效果优先**：高难度推理任务（如数学证明）→ GPT-4 / Claude-3-Opus；
  - **成本优先**：简单问答、格式化输出 → 轻量级模型（如 GPT-3.5 / 文心一言 Speed 版）；
  - **合规优先**：涉及敏感教育内容 → 国内备案模型；
  - **垂直优先**：数学解题 → 私有化数学专用模型；代码生成 → CodeLlama / Copilot。
- **动态调整**：基于线上 A/B 测试结果与用户反馈，自动更新路由权重。

#### 4.1.2 负载均衡（Load Balancer）
- **职责**：在同类模型的多个实例或 API Key 之间分配请求，避免单点限流。
- **策略**：轮询 + 权重 + 熔断（某 Key 触发速率限制时自动切换）。

#### 4.1.3 流控与降级（Rate Limiting & Fallback）
- **流控**：按租户、用户等级设置 QPS / TPM（Token Per Minute）上限。
- **降级策略**：
  - 一级降级：切换至同能力备选模型；
  - 二级降级：返回缓存结果（如常见知识点解释）；
  - 三级降级：提示"服务繁忙，请稍后重试"并进入异步队列。

#### 4.1.4 成本监控（Cost Monitor）
- **职责**：实时统计各模型、各智能体、各用户的 Token 消耗与费用。
- **能力**：
  - 按任务类型分摊成本（如"答疑Agent 本月消耗 $120"）；
  - 异常消耗告警（如单用户单次请求消耗 > 10k Tokens）；
  - 成本预测与预算管控。

#### 4.1.5 结果缓存（Response Cache）
- **职责**：缓存高频、稳定的问题回答，减少重复调用。
- **策略**：
  - 精确匹配缓存：完全相同的问题直接返回；
  - 语义相似缓存：向量相似度 > 0.95 的问题返回缓存结果（需校验时效性）；
  - 缓存失效：知识库更新后自动清除相关缓存。

#### 4.1.6 安全过滤（Safety Filter）
- **输入过滤**：识别并拦截涉及作弊、代写作业、不良诱导等违规请求。
- **输出过滤**：
  - 事实性校验：对教育类事实（如历史年代、公式定理）进行知识库交叉验证；
  - 毒性检测：过滤歧视、暴力、不当内容；
  - 置信度标注：对不确定的答案附加"仅供参考，建议核实"提示。

### 4.2 文本模型池（LLM Pool）

| 模型 | 适用场景 | 接入方式 |
|-----|---------|---------|
| GPT-4o / Claude-3.5-Sonnet | 复杂推理、长上下文理解、高质量生成 | API |
| GPT-3.5-Turbo / 文心一言 Speed | 日常问答、格式化输出、低成本场景 | API |
| 通义千问 / 智谱 GLM-4 | 国内合规场景、中文优化需求 | API |
| 私有化 LLaMA-3 / Qwen-72B | 高敏感数据、离线环境、深度定制 | 私有部署 |

**统一管理接口**：
```typescript
interface LLMRequest {
  model: string;           // 指定模型或 "auto" 自动路由
  messages: ChatMessage[]; // 对话历史
  temperature?: number;    // 创造性控制
  maxTokens?: number;      // 输出上限
  tools?: ToolDefinition[]; // 可选工具调用（Function Calling）
  stream?: boolean;        // 是否流式返回
}

interface LLMResponse {
  content: string;
  toolCalls?: ToolCall[];
  usage: { promptTokens: number; completionTokens: number };
  modelUsed: string;
  latencyMs: number;
}
```

### 4.3 多模态模型池（Multimodal Pool）

| 能力 | 模型/服务 | 适用场景 |
|-----|----------|---------|
| 图像理解 | GPT-4V / Claude-3-Opus / Qwen-VL | 拍照搜题、手写笔记识别、图表解析 |
| 图像生成 | Stable Diffusion / DALL-E 3 | 生成教学插图、概念可视化、记忆卡片 |
| 语音转文字 | Whisper / 讯飞语音识别 | 语音提问、课堂录音转笔记 |
| 文字转语音 | Azure TTS / 讯飞语音合成 | 文章朗读、听力材料生成、陪伴 Agent 语音交互 |
| 视频理解 | 自研视频抽帧 + VLM | 教学视频内容摘要、关键帧提取 |

**多模态请求统一封装**：
```typescript
interface MultimodalRequest {
  type: 'image-understanding' | 'image-generation' | 'speech-to-text' | 'text-to-speech';
  input: string | Buffer | URL;  // 文本、二进制文件或资源地址
  context?: string;              // 附加上下文（如"这是一道高中物理力学题"）
  outputFormat?: string;         // 期望输出格式
}
```

### 4.4 垂直领域模型（Domain-Specific Models）

#### 4.4.1 数学解题模型
- **用途**：专门处理代数、几何、微积分等数学问题的逐步推理。
- **实现**：基于开源数学模型（如 DeepSeek-Math、MetaMath）微调，或调用 Wolfram Alpha API 进行结果校验。
- **特色能力**：生成 LaTeX 格式推导过程、识别图形题、自动绘制函数图像。

#### 4.4.2 代码生成与评测模型
- **用途**：编程教学中的代码生成、代码解释、自动评测。
- **实现**：CodeLlama / StarCoder + 自研测试用例执行沙箱。
- **特色能力**：支持 10+ 编程语言、自动捕获运行时错误、给出修复建议。

#### 4.4.3 作文批改模型
- **用途**：语文/英语作文的评分、评语生成、改进建议。
- **实现**：基于教育领域标注数据微调的大模型 + 作文评分 rubric 引擎。
- **特色能力**：从立意、结构、语言、卷面多维度评分，生成个性化修改示例。

## 五、提示工程体系（Prompt Engineering）

### 5.1 提示模板管理
- **中央化存储**：所有提示词统一存储于版本控制系统，支持 A/B 测试与热更新。
- **分层设计**：
  - **系统提示（System Prompt）**：定义 Agent 角色、行为边界、输出格式；
  - **任务提示（Task Prompt）**：具体任务描述与约束；
  - **动态提示（Dynamic Prompt）**：运行时注入的学生画像、知识库片段、历史记忆。

### 5.2 提示优化策略
- **少样本学习（Few-shot）**：在提示中嵌入 2-3 个高质量示例，引导模型输出符合预期的格式。
- **思维链（Chain-of-Thought）**：要求模型先展示推理过程，再给出最终答案，提升复杂任务准确率。
- **自一致性（Self-Consistency）**：对同一问题采样多次，取多数一致答案，降低幻觉风险。
- **反思机制（Reflection）**：让模型检查自己的输出并修正错误（"请检查上述推导是否有误"）。

## 六、输出质量控制

### 6.1 后处理流水线
```
原始输出 → 格式校验 → 事实校验 → 安全过滤 → 置信度评分 → 最终输出
```

- **格式校验**：确保 JSON、Markdown、LaTeX 等格式合法，可正常渲染。
- **事实校验**：对教育事实类陈述，与知识库进行比对；对数学/代码结果，通过计算引擎验证。
- **安全过滤**：二次过滤模型输出的不当内容。
- **置信度评分**：模型对答案的确定程度量化，低置信度时触发人工复核或附加免责声明。

### 6.2 幻觉检测与缓解
- **检索增强生成（RAG）**：优先基于知识库检索结果生成回答，减少模型"编造"。
- **溯源标注**：回答中涉及的事实需标注来源（如"根据高中数学必修一第 3 章..."）。
- **不确定性表达**：鼓励模型在不确定时使用"可能""据我所知"等措辞，避免绝对化断言。

## 七、与上下层交互

```
智能体层  <--统一 API/SDK-->  模型层（请求推理与生成）
模型层   <--API / 私有部署-->  外部模型服务商 / 自研模型集群
模型层   <--向量检索 / SQL-->  数据层（知识库检索用于 RAG、学生画像用于个性化提示）
```
