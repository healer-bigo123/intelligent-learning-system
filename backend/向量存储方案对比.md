# 向量存储方案对比

## 两种方案的核心区别

| 特性 | 简化版（当前） | 完整版（Chroma + Embedding） |
|-----|-------------|---------------------------|
| **底层技术** | SQLite + 关键词匹配 | Chroma + BGE 语义向量 |
| **匹配方式** | 关键词重叠（Jaccard） | 语义向量余弦相似度 |
| **外部依赖** | 无（纯本地） | HuggingFace 模型下载 |
| **网络要求** | 不需要联网 | 首次需下载模型 |
| **启动速度** | 秒级 | 需等待模型加载 |
| **理解能力** | 字面匹配 | 语义理解（同义词、近义词） |

---

## 对项目功能的影响分析

### 1. 知识库上传 ✅ 无影响
- 两种方案都支持文档上传、分块、存储
- 简化版甚至更快（无需 Embedding 计算）

### 2. 检索功能 ⚠️ 部分影响

| 查询场景 | 简化版表现 | 完整版表现 |
|---------|----------|----------|
| "什么是人工智能" | ✅ 能匹配含"人工智能"的文档 | ✅ 同样能匹配 |
| "AI 是什么" | ⚠️ 可能匹配不到（无"AI"关键词） | ✅ 能理解 AI = 人工智能 |
| "神经网络原理" | ✅ 匹配含"神经网络"的文档 | ✅ 匹配更准确 |
| "怎么学机器学习" | ⚠️ 匹配"机器学习"文档 | ✅ 还能匹配"深度学习"相关文档 |

**结论**：
- 用户查询用词与文档用词一致时 → **无差别**
- 用户使用同义词、近义词时 → **完整版更优**

### 3. RAG 回答质量 ⚠️ 轻微影响

RAG 流程：
```
用户提问 → 检索相关文档 → 大模型生成回答
```

- **检索阶段**：简化版可能漏掉语义相关但关键词不同的文档
- **生成阶段**：无影响（大模型能力不变）

**实际影响**：
- 如果检索到 3 篇相关文档 vs 5 篇 → 大模型回答质量差异很小
- 如果完全没检索到 → 大模型只能基于自身知识回答

### 4. 评分/评估功能 ✅ 无影响

项目中的评分功能（如学习效果评估、资源推荐评分）：
- 基于大模型的判断能力
- 与向量存储方式无关

**使用简化版** ✅

- 后续可随时切换

### 生产阶段（上线后）

- 只需改 1 行配置切换

## 如何切换（未来）

当网络问题解决后：

```bash
# 1. 设置 HuggingFace 镜像
python set_hf_mirror.py

# 2. 下载模型（首次）
python test_chroma.py

# 3. 修改配置，启用完整版
# 在 .env 中添加：
# VECTOR_STORE_MODE=chroma

# 4. 重启服务
python main.py
```

**无需改动业务代码**，向量存储层自动切换。
